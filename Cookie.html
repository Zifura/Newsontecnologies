  1. Sobre Arduino

    2. O que é Arduino?

    Arduino é uma plataforma eletrônica de código aberto baseada em hardware e software fáceis de usar. As placas Arduino são capazes de ler entradas – luz em um sensor, um dedo em um botão ou uma mensagem do Twitter – e transformá-las em uma saída – ativando um motor, ligando um LED, publicando algo online. Você pode dizer à sua placa o que fazer enviando um conjunto de instruções ao microcontrolador da placa. Para isso utiliza-se a linguagem de programação Arduino (baseada em Wiring) e o Software Arduino (IDE), baseado em Processamento.

    Quais são os benefícios de Arduino?

    Um dos grandes benefícios é que é open-source, ou seja, a licença é gratuita sendo possivel montar, modificar, melhorar e personalizar qualquer projeto básico de hardware. Arduino possiblita a construção de uma linguagem própria e de desenvolver projetos autorais e o Arduino pode ser utilizado sem o auxílio de um computador. Uma vez que o programa é instalado na placa, ele será repetido em loop, necessitando apenas de uma fonte de alimentação.

    • Fácil de aprender e usar. A linguagem de programação do Arduino é baseada em C++, com uma sintaxe simples e direta, fácil de aprender até mesmo para novatos.
    • Versatilidade.
    • Amplamente suportado.
    • Custo efetivo.
    • Open Source.

    HTML5 é a ultima especificação da lingua HTML, e representa uma grande descoberta com as previas praticas de markup. HTML5 traz grandes mudanças para o design da lingua nomeadamente o encouragamento de markup semantico, separando design do conteudo, promover responssividade de acesso e design, redusindo o overlap entre HTML, CSS, e JavaScript e supportar media rica entre experiencias necessárias para plugins tais como o Flash ou Java.

    A maioria das mudanças individuais é resultado de objetivos maiores no design da linguagem. Esses objetivos incluem principalmente:

    • Incentivando a marcação semântica (significativa).
    • Separando design de conteúdo.
    • Promover a acessibilidade e a capacidade de resposta do design.
    • Reduzindo a sobreposição entre HTML, CSS e JavaScript.
    • Oferece suporte a experiências de rich media e elimina a necessidade de plug-ins como Flash ou Java

    Dominar o HTML5 não envolve apenas aprender quais recursos CSS substituem os antigos recursos do HTML. Se você deseja ter uma noção intuitiva do HTML5, é melhor entender como esses objetivos afetaram o desenvolvimento da linguagem.

Tanto os reguladores como os grupos industriais como o IAB expressaram preocupações sobre as potenciais desvantagens e limitações competitivas do Privacy Sandbox, sugerindo que o Google pode precisar de resolver estas questões antes de prosseguir.

    Apesar dos desafios e das críticas, o Google continua comprometido com as tecnologias Privacy Sandbox, enfatizando o seu objetivo de melhorar a privacidade e, ao mesmo tempo, manter a publicidade direcionada.     A proposta de Java vai dar uma excepção para switch

    A legibilidade, facilidade de manutenção e usabilidade de instruções e expressões switch em Java seriam melhoradas por uma proposta para permitir que exceções sejam tratadas no bloco switch.

    A proposta atual do OpenJDK, “Tratamento de exceções no switch (visualização)”, faria parte da Standard Edition do Java, embora nenhuma versão específica do Java SE tenha sido designada ainda como destinatário. Especificamente, a proposta pede o aprimoramento do switch para que as exceções lançadas pelo seletor (o e no switch (e)…) possam ser tratadas no bloco switch.

    As metas do plano, que foi criado em 12 de janeiro e atualizado em 19 de abril, incluem melhorar a legibilidade e a capacidade de manutenção, permitindo que o switch lide de forma concisa com todos os resultados possíveis da avaliação do seletor e simplificando o uso de APIs que lançam exceções verificadas, quando usadas pelo seletor. de uma instrução ou expressão switch.

    O que motiva a proposta é a noção de que aprimorar a opção para lidar com exceções do seletor agilizará o uso de bibliotecas que lançam exceções, melhorará a legibilidade e reduzirá erros. Um switch incorpora uma escolha multidirecional baseada no valor de uma expressão seletora.

    Tradicionalmente, a mudança tem sido hostil às exceções, afirma a proposta. Se a expressão do seletor gerou uma exceção em vez de produzir um valor, switch relançou a exceção, dificultando o uso do switch. Os desenvolvedores foram forçados a lidar com exceções do seletor, normalmente colocando o switch em um bloco try-catch, uma abordagem desajeitada que tem desvantagens que levam a programas piores. Uma abordagem muito melhor seria permitir que os desenvolvedores tratassem uma exceção do seletor com apenas outro caso, conclui a proposta.

    Não é um objetivo da proposta tratar exceções lançadas pelo bloco switch no bloco switch, ou introduzir novos tipos de padrões que correspondam a exceções ou incorporar suporte para tratamento de exceções em outras instruções e expressões. Nem é objetivo alterar o modelo de exceções verificadas versus exceções não verificadas, seja em geral ou no âmbito de uma mudança, diz a proposta.

    The Python Language Summit 2024: anotações como transformadores

    Palestra final da programação principal do Python Language Summit foi proferida por Jason R. Coombs sobre o uso de anotações para transformações. A apresentação foi acompanhada por um repositório GitHub e um notebook Jupyter ilustrando o problema e a solução proposta.

    Jason está interessado em um método para os usuários "transformarem seus parâmetros de forma reutilizável". A motivação era evitar métodos imperativos de transformação de parâmetros para "aumentar a reutilização, composição e separação de interesses". Jason imaginou transformadores que poderiam ser "empacotados em uma biblioteca ou usados em múltiplas funções" e "seriam aplicados no escopo de parâmetros individuais".

    Python já possui um recurso de linguagem semelhante a esse conceito com decoradores, que permite agrupar uma função ou classe com outra função de forma sintaticamente concisa.

    Jason observou que "os valores de retorno podem ser manipulados pelos decoradores com bastante facilidade, então [a proposta] está mais preocupada com os parâmetros de entrada". Para que um decorador afete os parâmetros, o decorador "teria que inspecionar os parâmetros" e "se envolver com a assinatura da função".

  6. Sobre C

    7. O que é C?

    C é uma linguagem processual imperativa, suportando programação estruturada, escopo de variável léxica e recursão, com um sistema de tipo estático. C foi projetado para ser compilado para fornecer acesso de baixo nível à memória e construções de linguagem que mapeiam eficientemente as instruções da máquina, tudo com suporte mínimo de tempo de execução.

    Quais são os beneficios de C?

    C é uma linguagem muito eficiente que pode escrever código rápido e confiável. C é uma linguagem portátil, o que significa que o código escrito em C pode ser facilmente compilado e executado em várias plataformas.

    • É fácil de entender. Uma das principais razões pelas quais as pessoas escolhem C em vez de outras linguagens de programação é a sua simplicidade.
    • Presença de muitas bibliotecas.
    • Fácil de escrever.
    • Baixo custo.
    • Velocidade de execução rápida.
    • Portátil.
    • Depuração fácil.
    • Linguagem Orientada a Procedimentos.
